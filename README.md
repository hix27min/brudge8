# Nekto Audio Bridge (hosted)

Веб-версия моста для Nekto.me. Работает на сервере, а ты заходишь с телефона/ПК через браузер.

```
Собеседник A ──nekto──┐
                      ├──→ Сервер (Python) ←──WebRTC──→ Твой телефон/ПК (браузер)
Собеседник B ──nekto──┘         ↑
                             микрофон берётся с браузера
                             и стримится на сервер
```

---

## ⚠️ Важно

1. Нужен HTTPS (чтобы браузер разрешил микрофон). Все бесплатные хостинги из списка ниже автоматически дают HTTPS.
2. Для каждого аккаунта nekto.me надо пройти **3 капчи** в голосовом чате (в браузере, до копирования токена).
3. User-Agent обязательно **тот же**, в котором проходил капчи.
4. Два разных аккаунта nekto (не один!).

## Бесплатные хостинги для этого

Подходят те, что дают Python/Docker, WebSocket и **исходящий UDP** (для WebRTC). С WebSocket ок у всех, с UDP — у многих хуже. В порядке простоты/надёжности:

### 🥇 Fly.io (рекомендую)
- Бесплатный tier: 3 маленьких VM (256MB RAM), $5 кредита
- Поддерживает WebRTC (UDP), Docker
- Свой домен `*.fly.dev` c HTTPS
- Минус: требует привязки карты
- URL: https://fly.io

### 🥈 Render.com
- Бесплатный web-сервис, засыпает после 15 минут без трафика
- Поддерживает Docker
- HTTPS из коробки
- Минус: WebRTC иногда работает нестабильно из-за ограничений на UDP
- URL: https://render.com

### 🥉 Railway.app
- $5/месяц бесплатного кредита (достаточно для одного лёгкого сервиса)
- Docker, HTTPS
- Минус: после лимита надо платить
- URL: https://railway.app

### Платные, но дёшево (надёжнее всего):
- **Hetzner Cloud** — €4.51/мес за виртуалку 2GB RAM, европейский дата-центр
- **Contabo VPS S** — €4.5/мес
- **Oracle Cloud Free Tier** — совершенно бесплатно, 4 ARM виртуалки по 6GB RAM (навсегда), но иногда сложно зарегистрироваться (нужна карта)

### Что ⚠️ НЕ подойдёт:
- PythonAnywhere free (нет WebSocket)
- Heroku (нет фри тира, UDP ограничен)
- Replit (UDP ограничен)

## Деплой на Fly.io (5 минут)

1. Создай аккаунт и установи `flyctl`:
   ```
   curl -L https://fly.io/install.sh | sh
   flyctl auth signup
   ```

2. Положи этот репозиторий куда-нибудь и запусти:
   ```
   flyctl launch
   ```
   Соглашайся на всё по умолчанию, имя приложения придумай (например `my-nekto`).

3. `flyctl deploy`

4. Открой `https://my-nekto.fly.dev` в браузере телефона.

5. Нажми **"Включить микрофон"** — браузер запросит доступ, дай.

6. Вставь два токена (каждый из своего аккаунта), два UA, жми Подключить в A и B.

---

## Деплой на Render.com

1. Форкни (или залей) этот код в GitHub-репозиторий.

2. На Render: **New → Web Service → Connect GitHub repo**.

3. Настройки:
   - Environment: Docker
   - Region: любой
   - Plan: Free

4. Deploy. Получишь URL типа `https://my-nekto.onrender.com`.

5. Открой в браузере, дай доступ к микрофону, вставь токены.

**Проблема Render**: может «засыпать» после 15 мин неактивности. Первый запрос после паузы грузится 30 секунд. Для постоянной работы — платный тариф.

## Локальный запуск (чтобы сначала проверить)

```bash
pip install -r requirements.txt
python nekto_bridge_hosted.py --port 8765
```

Открывай `http://localhost:8765`. На `localhost` браузер разрешит микрофон без HTTPS. На удалённом сервере — только HTTPS.

## Как это работает

1. **Ты заходишь в браузере на сервер.** Страница загружается через HTTPS.
2. **Нажимаешь "Включить микрофон".** Браузер:
   - запрашивает доступ к микрофону
   - создаёт WebRTC peer-connection с сервером
   - шлёт свой микрофон серверу
   - получает обратно смешанный звук (голоса A+B)
3. **Сервер** держит два независимых WebSocket'а к `audio.nekto.me` (отдельно для каждого аккаунта A и B), ведёт свои разговоры с двумя случайными собеседниками.
4. **Аудио-маршрутизация на сервере:**
   - голос A → шлётся в канал B (собеседник B слышит собеседника A)
   - голос B → шлётся в канал A (собеседник A слышит собеседника B)
   - голос A + голос B → шлётся в твой браузер (ты слышишь обоих)
   - твой микрофон → шлётся в каналы A и/или B (по твоему выбору)

## Безопасность

- Токены nekto хранятся в `localStorage` твоего браузера
- На сервер они попадают только когда ты жмёшь "Подключить" (через WebSocket), и живут в оперативке, не сохраняются
- Если сервер публичный и кто-то угадает URL — он сможет использовать твой мост. Добавь HTTP Basic Auth через nginx/Caddy если параноишь.
